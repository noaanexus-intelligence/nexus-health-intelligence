Security & Governance

ยึดหลัก aggregated data, privacy by design และการกำกับดูแลข้อมูลที่เหมาะสม

Problem

ข้อมูลสุขภาพมีมาก แต่ใช้บริหารได้ยาก

ทีมงานในพื้นที่มีข้อมูลครบ แต่การแปลงข้อมูลให้เป็นการตัดสินใจยังเป็นงานหนักทุกเดือน

รายงาน HDC จำนวนมาก ดูภาพรวมได้ยาก

HDC มีรายงานจำนวนมาก แต่ดูภาพรวมระดับอำเภอได้ยาก ต้องไล่เปิดทีละหมวด

KPI กระจายหลายหมวด

KPI และ Service Plan กระจายอยู่หลายหมวด ทำให้ติดตามผลรวมในภาพเดียวได้ยาก

ต้องเปิดหลายหน้าจอ

ผู้บริหารต้องเปิดหลายหน้าจอเพื่อดูข้อมูลรายหน่วยบริการ เสียเวลาในการรวบรวม

ไม่ทราบล่วงหน้าว่าตัวชี้วัดใดมีความเสี่ยง

ไม่สามารถทราบได้ในทันทีว่าตัวชี้วัดใดมีความเสี่ยงไม่บรรลุเป้าหมาย จนกว่าจะใกล้สิ้นปีงบประมาณจึงจะทราบ

ข้อมูล 43 แฟ้มผิดพลาด

ความผิดพลาดของข้อมูล 43 แฟ้ม อาจทำให้ผลการดำเนินงานที่ปรากฏต่ำกว่าความเป็นจริง

ขาดระบบสรุป Insight

ขาดระบบสรุป insight สำหรับการประชุมและติดตามงาน ต้องทำสไลด์ใหม่ทุกครั้ง

Solution

Health Data Intelligence Platform สำหรับสาธารณสุขระดับพื้นที่

ระบบรวมข้อมูลจาก HDC, KPI Template, Service Plan และข้อมูลรายหน่วยบริการเข้าด้วยกันในหน้าเดียว เพื่อช่วยให้ผู้บริหารดูภาพรวม วิเคราะห์ Gap จัดอันดับหน่วยบริการ และติดตามคุณภาพข้อมูลได้รวดเร็วขึ้น

  1. 1. รวบรวมข้อมูล

    ดึงรายงาน HDC, 43 แฟ้ม, FDH/eClaim, KPI Template และ Service Plan มาไว้ที่เดียว

  2. 2. จัดโครงสร้าง

    ทำ Crosswalk และจัดหมวดให้ข้อมูลเชื่อมโยงกัน พร้อมระบบ Snapshot

  3. 3. วิเคราะห์

    คำนวณ Gap, Trend และ Ranking รายหน่วยบริการ พร้อมเตือนคุณภาพข้อมูล

  4. 4. นำเสนอ Insight

    สรุปเป็น Executive Brief และ Dashboard ที่พร้อมใช้ในการประชุมติดตามงาน

Data Security & Governance

ออกแบบโดยคำนึงถึง Data Security และ Privacy

ปกป้องข้อมูลสุขภาพตามหลัก privacy และมีแนวทางกำกับดูแลข้อมูล โดยไม่เปิดเผยข้อมูลผู้ป่วยรายบุคคล

Data Security

  • ใช้ข้อมูล aggregated data
  • ไม่แสดง CID, HN หรือชื่อผู้ป่วย
  • ไม่แสดงข้อมูลผู้ป่วยรายบุคคล
  • ใช้ HTTPS และ Security Headers
  • ใช้ Snapshot เพื่อลดการเรียก API โดยไม่จำเป็น

Data Governance

  • กำหนดสิทธิ์การเข้าถึงตามบทบาท (สสอ./CUP/หน่วยบริการ)
  • แยกข้อมูลตามขอบเขตพื้นที่ที่รับผิดชอบ
  • แนวทางคุณภาพและความถูกต้องของข้อมูล
  • รองรับ Audit Log และ Monitoring ในอนาคต
  • นโยบายการเก็บและใช้ข้อมูลที่โปร่งใส
